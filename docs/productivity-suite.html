🍅 Focus Timer (Pomodoro)

A full Pomodoro timer built into the chat interface. Cycles through work → short break → work → … → long break automatically. Configurable durations, audio cues, and session counting help you maintain deep focus while chatting with AI.

Shortcut: Alt+P

Interactive Demo — Try It

25:00
Work

How It Works

PhaseDefault DurationRing Color
Work25 minutesBlue
Short Break5 minutesGreen
Long Break (every 4 sessions)15 minutesGreen

API

FocusTimer.toggle()       // Open/close the panel (Alt+P)
FocusTimer.start()        // Start the current phase
FocusTimer.pause()        // Pause the countdown
FocusTimer.reset()        // Reset to idle state
FocusTimer.skip()         // Skip to next phase
FocusTimer.getState()     // → 'idle' | 'work' | 'short-break' | 'long-break'
FocusTimer.configure({    // Update durations (minutes)
  work: 50,
  short: 10,
  long: 20,
  longAfter: 4            // Long break after N work sessions
})

Persistence

Timer state (current phase, remaining time, sessions completed) is saved to localStorage under the key ac_focus_timer. If you close the tab mid-session, the timer resumes approximately where you left off, accounting for elapsed wall-clock time.

📋 Conversation Agenda

Set goals before you start chatting. The agenda panel tracks per-session objectives with a visual progress bar — so you know when you've accomplished what you set out to do.

Up to 50 goals per session, 200 characters each.

Interactive Demo — Build Your Agenda

    0 / 0 complete 0%

    API

    ConversationAgenda.toggle()         // Open/close panel
ConversationAgenda.add('Fix CORS')  // Add a goal programmatically
ConversationAgenda.list()           // → [{ id, text, done, createdAt }]
ConversationAgenda.progress()       // → { total, done, pct }

    Design Decisions

    • Goals are scoped to the current session — switching sessions loads that session's agenda
    • Completed goals sort to the bottom automatically
    • Progress bar uses a gradient from blue → green as completion increases
    • Storage key: ac-agenda (namespaced by session ID internally)

    Data Collection

    The heatmap aggregates message timestamps from all saved sessions. Each time you send a message, the current hour and day-of-week are incremented in a 7×24 matrix stored in memory. The module reads session data on-demand rather than maintaining a separate persistent counter.

    🎵 Ambient Sound Player

    Procedurally generated ambient soundscapes via the Web Audio API — no audio files needed. Each sound is synthesized in real-time using oscillators, noise generators, and filters. Mix multiple sounds at custom volumes.

    Shortcut: Alt+A

    Interactive Demo — Sound Mixer (Visual Only)

    Click sounds to toggle, adjust sliders to mix. (No actual audio — demonstrates the UI pattern.)

    Sound Generation Techniques

    SoundMethodParameters
    🌧 RainFiltered white noise + random droplet impulsesBandpass 200–2000 Hz, density modulation
    ☕ CaféBrown noise + layered low-frequency murmurLowpass 800 Hz, occasional clink impulses
    🔥 FireFiltered noise with crackling envelopeBandpass 100–1500 Hz, random amplitude spikes
    💨 WindModulated filtered noiseLFO on filter cutoff (0.1–0.5 Hz sweep)
    🏞 StreamWhite noise + resonant bandpassMultiple resonant peaks, gentle modulation
    📻 White NoiseRaw white noise bufferFlat spectrum, gain control only

    Architecture

    Each sound creates its own GainNode connected to a shared masterGain node. Toggling a sound creates or destroys its node chain. Volume sliders map to individual gain values (0.0–1.0). AudioContext is created lazily on first user interaction to comply with browser autoplay policies.

    ⌨️ Command Palette

    A VS Code-style universal command launcher. Press Ctrl+Shift+P to open a fuzzy-search overlay that provides instant access to every feature in the app.

    Interactive Demo — Fuzzy Search

    Fuzzy Matching Algorithm

    The palette uses a character-by-character subsequence match with scoring bonuses for:

    • Consecutive matches — characters in sequence score higher
    • Word-boundary matches — matching the first letter after a space/separator gets a bonus
    • Case-exact matches — matching the exact case adds a small bonus

    Results are sorted by fuzzy score descending. The selected item is highlighted and navigable with ↑/↓ arrows and executed with Enter.

    Registering Commands

    // Any module can register commands at init time:
CommandPalette.register({
  id: 'my-feature-toggle',
  label: 'Toggle My Feature',
  icon: '🔧',
  shortcut: 'Alt+M',
  action: () => MyFeature.toggle()
});

// Commands support categories for grouping:
CommandPalette.register({
  id: 'export-json',
  label: 'Export Conversation as JSON',
  icon: '📥',
  category: 'Export',
  action: () => HistoryPanel.exportJSON()
});

    ⏱ Conversation Timer

    Tracks active time spent in each conversation. Auto-pauses after 2 minutes of inactivity (no keypresses, no scrolling) and resumes when you interact again. Shortcut: Alt+T

    How Active Time Differs from Wall-Clock Time

    If you open a chat at 2:00 PM, chat actively until 2:15 PM, step away for 30 minutes, then return and chat for 10 more minutes — the wall clock says 55 minutes, but the Conversation Timer reports 25 minutes of active time.

    Features

    • Auto-pause: 120-second idle timeout (configurable)
    • Per-session tracking: Each session has its own accumulated time
    • Time log dashboard: View historical active time across all sessions
    • Persistence: Stored in ac_conversation_timer via SafeStorage

    API

    ConversationTimer.toggle()       // Open/close dashboard (Alt+T)
ConversationTimer.getElapsed()   // → milliseconds of active time this session
ConversationTimer.isRunning()    // → boolean
ConversationTimer.reset()        // Zero out current session timer

    💾 Draft Recovery

    Never lose an unsent message again. DraftRecovery auto-saves the chat input to localStorage every 500ms (debounced). When you reload or switch sessions, your unsent text is restored automatically with a toast notification.

    How It Works

    1. On every input event, a 500ms debounce timer starts
    2. When the timer fires, the current input text is saved to agenticchat_drafts keyed by session ID
    3. On page load or session switch, any saved draft for the active session is restored into the input field
    4. Successfully sending a message clears the draft for that session

    API

    DraftRecovery.save()        // Force-save current input immediately
DraftRecovery.restore()     // Restore draft for current session
DraftRecovery.clear()       // Clear draft for current session
DraftRecovery.hasDraft()    // → boolean
