📌 Smart Fact Memory

An autonomous knowledge extractor that scans every AI response for facts, decisions, preferences, action items, and definitions — building a persistent cross-session knowledge base you can search and reference later. Shortcut: Alt+Shift+F

    Extraction Patterns

    The module uses 12 regex patterns organized by category to identify knowledge worth remembering:

    CategoryTrigger PhrasesExample
    ✅ Actionyou should, remember to, TODO:, - [ ]"Remember to update the API docs before release"
    ⚖️ DecisionI've decided, let's go with, chosen approach"Let's go with PostgreSQL for the main database"
    💡 PreferenceI prefer, I always use, best way to"I prefer using async/await over raw promises"
    📖 Definition**Term** — ..., X means, X refers to"**SSR** — Server-Side Rendering"
    📌 Factnote that, key takeaway, the default is"Note that the maximum payload size is 10MB"

    API

    SmartFactMemory.toggle()              // Open/close panel (Alt+Shift+F)
SmartFactMemory.extractFacts(text)    // → [{ category, text, label? }]
SmartFactMemory.addFacts(factsArray)  // Persist extracted facts
SmartFactMemory.getRelevantFacts(query) // TF-IDF search across stored facts
SmartFactMemory.facts()               // → all stored facts (copy)
SmartFactMemory.count()               // → number of stored facts
SmartFactMemory.isEnabled()           // → boolean
SmartFactMemory.setEnabled(bool)      // Enable/disable auto-extraction

    Storage & Limits

    • Facts stored in sfm_facts via SafeStorage (respects Incognito Mode)
    • Maximum 2,000 facts — oldest evicted when limit reached
    • Each fact records: category, text, optional label, source session ID, timestamp, pinned state
    • Deduplication via normalized text comparison prevents storing the same fact twice

    💡 Recommendations

      Budget Analysis

      The module computes a comprehensive snapshot of token usage:

      MetricDescription
      totalTokensEstimated tokens across all messages in context
      maxTokensModel's context window limit (e.g., 128K for GPT-4o)
      usagePctPercentage of context consumed
      breakdownPer-role token counts (system, user, assistant)
      avgTokensPerExchangeAverage tokens per user↔assistant exchange
      exchangesUntilFullPredicted exchanges remaining before hitting limit
      codePctPercentage of tokens that are code blocks
      recommendationsContextual compression suggestions

      Compression Recommendations

      Based on usage patterns, the module suggests actions like:

      • Summarize old messages — when context is >70% full with many exchanges
      • Extract code to snippets — when code blocks consume >30% of context
      • Start a new session — when context is >90% full
      • Use a larger-context model — when you frequently hit limits

      🔮 Smart Model Advisor

      Analyzes your prompt before you send it and recommends the optimal model based on task complexity, code content, creativity needs, and cost sensitivity. Shortcut: Alt+Shift+A

      Interactive Demo — Task Analysis

      Task Profiles

      ProfileRecommended ModelSignals
      🧑‍💻 Code GenerationGPT-4oCode keywords, backticks, programming language names
      🎨 Creative WritingGPT-4oStory, poem, creative, imagine, write (longer output)
      📊 AnalysisGPT-4oAnalyze, compare, evaluate, pros/cons, trade-offs
      💬 Quick ChatGPT-4o-miniShort questions, simple facts, greetings
      📝 SummarizationGPT-4o-miniSummarize, TL;DR, key points, brief
      🔧 Debug/FixGPT-4oBug, error, fix, stack trace, segfault, exception

      Scoring Algorithm

      Each prompt is scored across four dimensions (0–100):

      • Complexity — word count, sentence structure, technical terminology density
      • Code Content — presence of code blocks, language keywords, syntax patterns
      • Creativity — creative writing signals, open-ended phrasing, imagination cues
      • Specificity — concrete details, constraints, format requirements

      The highest-scoring task profile determines the recommendation. Confidence is computed from how clearly one profile dominates the others.

      😊 Conversation Sentiment

      Heuristic sentiment analysis that tracks the emotional tone of your conversation over time. Each message is scored as positive, negative, or neutral using a built-in word lexicon. Shortcut: Ctrl+Shift+M

      Interactive Demo — Mood Timeline

      Neutral

      Lexicon-Based Scoring

      The module maintains two word sets:

      Positive words (60+): good, great, excellent, amazing, wonderful, fantastic, awesome, love, happy, joy, perfect, brilliant, beautiful, helpful, thank, appreciate, solved, works, success, progress, …

      Negative words (60+): bad, terrible, awful, horrible, broken, error, fail, crash, bug, frustrating, confused, wrong, problem, issue, stuck, slow, hate, annoying, difficult, mess, …

      Each message's score = (positive_count − negative_count) / total_scored_words, normalized to [-1, +1]. Messages with no scored words default to neutral (0).

      Mood Shift Detection

      The ConversationMoodRing (the companion module) watches for sudden sentiment shifts — e.g., a conversation that was positive suddenly turning negative. When detected, it surfaces an alert suggesting a break, rephrasing, or model switch.

      📋 Smart Conversation Digest

      Generates a cross-session digest that extracts topics, action items, unresolved questions, and sentiment from all recent sessions. Perfect for reviewing what you've been working on. Shortcut: Alt+Shift+J

      Sample Digest Output

      🏷 Topics Discussed
      Authentication PostgreSQL React Hooks CI/CD Pipeline Docker Networking TypeScript Generics
      ✅ Action Items

      • Add connection pooling to database layer (PgBouncer)
      • Write integration tests for the new webhook handler
      • Update API docs to reflect v3 breaking changes
      ❓ Unresolved Questions
      • Should we use Redis or Memcached for session storage?
      • What's the optimal batch size for the ETL pipeline?
      • Is server-side rendering worth the added complexity?
      📊 Session Sentiment Summary
      • Auth Refactor — 😊 Positive (problem solved successfully)
      • Docker Debug — 😤 Negative→Positive (struggled then found fix)
      • React Components — 😐 Neutral (exploratory discussion)

      How It Works

      1. Reads all sessions stored in SafeStorage (respects Incognito Mode — excluded sessions are skipped)
      2. Extracts topics using TF-IDF keyword scoring from TextAnalytics
      3. Identifies action items via regex patterns (same as SmartFactMemory's action category)
      4. Detects questions by scanning for sentences ending with ? that lack clear answers in subsequent messages
      5. Computes per-session sentiment using the ConversationSentiment lexicon

      Export Formats

      The digest can be exported as Markdown or JSON for sharing with teammates or pasting into project docs.

      📈 Smart Session Insights

      A cross-session analytics dashboard with topic trends, model usage patterns, productivity metrics, and proactive recommendations. Shortcut: Alt+Shift+K

      Dashboard Sections

      📊 Topic Trends

      Word cloud and frequency chart showing what you've been discussing most over the past 7/30/90 days. Identifies emerging topics (increasing frequency) and fading ones.

      🤖 Model Usage

      Breakdown of which models you've used and for what task types. Surfaces opportunities to use cheaper models for simple tasks.

      ⏱ Productivity Patterns

      Average session length, messages per session, time-of-day patterns, and completion rates (agenda goals checked off vs. total).

      💡 Proactive Recommendations

      Suggestions based on your patterns: "You often debug late at night — consider GPT-4o for complex debugging", "Your TypeScript sessions average 2x longer than Python — try splitting into subtasks", etc.

      Data Sources

      Session Insights aggregates data from multiple modules:

      • SessionManager — session metadata, timestamps, message counts
      • ConversationTimer — active time per session
      • CostDashboard — model usage and API spend
      • ConversationSentiment — mood trends over time
      • ConversationAgenda — goal completion rates
      • ResponseRating — user satisfaction signals

      💰 Cost Dashboard

      Persistent API spend tracker with budget alerts, per-model breakdowns, and daily/weekly/monthly views. Every API call logs its token usage and estimated cost.

      Features

      • Real-time tracking: Every API request records input tokens, output tokens, model, and computed cost
      • Budget alerts: Set a monthly budget — get warned at 80% and blocked at 100% (configurable)
      • Per-model breakdown: See which models cost the most and optimize accordingly
      • Time filters: Today, this week, this month, all time
      • Export: CSV export for expense tracking or analysis

      🏗 Intelligence Module Architecture

      All Smart* modules follow the same architectural pattern:

      const SmartModule = (() => {
  // 1. Constants & storage keys
  const STORAGE_KEY = 'sm_module_data';

  // 2. Private state
  let _data = [];
  let _panel = null;
  let _visible = false;

  // 3. SafeStorage persistence (respects IncognitoMode)
  function _save() { SafeStorage.setItem(STORAGE_KEY, JSON.stringify(_data)); }
  function _load() { _data = JSON.parse(SafeStorage.getItem(STORAGE_KEY)) || []; }

  // 4. Processing logic (pattern matching, scoring, analysis)
  function _analyze(input) { /* ... */ }

  // 5. UI rendering (dynamic DOM, no framework)
  function _renderPanel() { /* ... */ }

  // 6. Public API (revealing module pattern)
  return {
    toggle: () => { _visible = !_visible; _visible ? _show() : _hide(); },
    show: _show,
    hide: _hide,
    // Module-specific methods...
  };
})();

      Cross-Module Communication

      Intelligence modules communicate through direct function calls rather than events:

      • SmartFactMemory uses TextAnalytics.tokenise() for text preprocessing
      • SmartSessionInsights reads from SessionManager, ConversationTimer, CostDashboard, and ConversationSentiment
      • SmartTokenBudgetManager uses ConversationManager.getHistory() and ChatConfig for model limits
      • All modules route through SafeStorage — ensuring Incognito Mode suppresses persistence globally
