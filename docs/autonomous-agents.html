🧭 Conversation Drift Detector

AutonomousAlt+Shift+D

Monitors topic coherence throughout a conversation by comparing TF-IDF cosine similarity between the opening anchor messages and a sliding recent window. When similarity drops below the alert threshold (default 0.35), the detector fires a toast notification and suggests refocusing prompts.

How It Works

  • 📌The first 4 messages form the anchor topic — their TF-IDF vector is cached once and never recomputed
  • 🔍The most recent 6 messages form the comparison window, re-tokenised each check
  • 📐Cosine similarity between anchor and window vectors gives a drift score (0 = total drift, 1 = on-topic)
  • 🔔When score < threshold, a non-intrusive alert with refocus suggestions appears

Configuration

ParameterDefaultDescription
alertThreshold0.35Cosine similarity below this triggers a drift alert
windowSize6Number of recent messages in the comparison window
anchorSize4Number of opening messages forming the anchor topic
autoAlertEnabledtrueWhether proactive toast alerts fire automatically

API

ConversationDriftDetector.toggle()           // open/close panel
ConversationDriftDetector.analyzeMessage()    // check latest message for drift
ConversationDriftDetector.analyzeConversation() // full conversation drift report

🚀 Conversation Momentum

Autonomous

Tracks engagement velocity in real-time with a 0–100 momentum score. The score rises with rapid, substantive exchanges and decays during idle periods. When momentum enters a "flow state" (sustained high score), the tracker celebrates; when it drops critically low, it nudges the user to re-engage.

Scoring Signals

  • ⬆️Message velocity: fast back-and-forth increases momentum
  • ⬇️Idle decay: score decays over time when no messages are sent
  • 🔥Flow detection: sustained score > 75 triggers "flow state" recognition
  • ⚠️Low alert: score < 20 triggers a gentle re-engagement suggestion

Tracked Statistics

MetricDescription
Peak ScoreHighest momentum reached in the session
Flow CountNumber of flow-state episodes detected
Total MessagesMessages sent since session start
Active TimeCumulative time the user has been actively engaged

API

ConversationMomentum.toggle()  // open/close momentum panel
ConversationMomentum.show()
ConversationMomentum.hide()

🎓 Conversation Coach

AutonomousAlt+Shift+H

Monitors user messages for common prompt anti-patterns and delivers non-intrusive coaching tips in real-time. Maintains a quality score, tracks improvement streaks, and provides periodic health summaries. Think of it as a writing coach that helps you get better AI responses.

Anti-Patterns Detected

  • ❌Vague prompts lacking specificity or context
  • ❌Overly broad requests without constraints
  • ❌Missing format expectations (code, list, essay, etc.)
  • ❌No role/persona specification when one would help

Coaching Mechanics

ParameterValueDescription
Tip cooldown3 messagesMinimum messages between coaching tips
Dismiss memory10 messagesWon't repeat the same tip type for this many messages
Auto-dismiss8 secondsTips disappear automatically after this duration
Health interval10 messagesMessages between health summary reports

Dashboard Metrics

The dashboard (Alt+Shift+H) shows: total messages analysed, good prompt percentage, current streak, best streak, average prompt length, and quality score distribution.

🌳 Conversation Brancher

AutonomousAlt+Shift+B

An autonomous intelligence layer built on top of ConversationFork. Tracks parent–child branch relationships, renders a visual branch tree, detects topic drift and proactively suggests forking, and offers side-by-side branch comparison.

Features

  • 🗂️Branch tree: visual hierarchy of all conversation branches with parent/child links
  • 🔀Drift-based fork suggestions: checks every 10 messages; if topic shifted, suggests creating a branch
  • ⚖️Side-by-side comparison: compare two branches to see how conversations diverged
  • 💾Persistent tree: branch metadata stored in SafeStorage with in-memory cache for O(1) lookups

API

ConversationBrancher.toggle()       // open/close branch panel
ConversationBrancher.recordBranch() // manually record a branch event
ConversationBrancher.checkDrift()   // trigger a drift check now
ConversationBrancher.onMessageAdded() // hook called on each new message

🧭 Conversation Compass

AutonomousAlt+Shift+NCanvas

A topic trajectory tracker that visualises where the conversation has been and where it's heading. Features an animated Canvas compass rose with a heading needle, a topic waypoint timeline, and proactive course-correction suggestions when drift or stagnation is detected.

Visualisation Components

  • 🎯Compass rose: animated Canvas 2D compass with a needle pointing toward the current topic vector
  • 📍Topic waypoints: timeline showing each major topic shift as a labelled waypoint
  • 📊Heading detection: TF-IDF cosine similarity across sliding windows determines direction
  • 🔄Auto-monitor: optional toast alerts when drift or stagnation is detected

Access

Keyboard: Alt+Shift+N  |  Slash: /compass  |  Command Palette: "compass"

📦 Conversation Stash

Ctrl+Shift+Z

Git-stash-style save/restore for conversations. Stash the current conversation to a LIFO stack, clear the chat, and start fresh. Pop the stash to restore a previous state. Supports multiple stash entries (up to 20) with descriptions and timestamps.

Workflow

// Stash current conversation (saves to stack, clears chat)
ConversationStash.push("WIP: debugging auth flow")

// Start a new conversation...

// Restore previous state
ConversationStash.pop()   // restores most recent stash

API

ConversationStash.toggle()  // open/close stash panel
ConversationStash.show()
ConversationStash.hide()

🧩 Smart Context Sidebar

AutonomousAlt+Shift+I

A proactive intelligence panel that automatically extracts and displays contextual information from the conversation: detected entities (URLs, code languages, file paths, numbers), topic-specific follow-up suggestions, conversation statistics, and contextual quick actions.

Entity Extraction

  • 🔗URLs and links mentioned in the conversation
  • 💻Programming languages detected from code blocks
  • 📁File paths referenced in messages
  • 🔢Notable numbers, versions, and measurements

Topic-Aware Follow-ups

The sidebar detects the current topic and offers relevant follow-up prompts. Supported domains include:

API

"How do I authenticate?" · "What are the rate limits?" · "Show error handling"

Database

"How do I handle migrations?" · "What about indexing?" · "Show a query example"

Security

"What are the vulnerabilities?" · "How do I sanitize input?" · "What about CORS?"

Performance

"Where are the bottlenecks?" · "Can you add caching?" · "What about lazy loading?"

Also supports: CSS, JavaScript, Python, React — each with tailored suggestions.

Implementation

Uses a MutationObserver on #chat-output to stay current without polling. Panel visibility state is persisted in SafeStorage.

🕸️ Smart Knowledge Map

AutonomousAlt+Shift+KCanvas

An autonomous knowledge graph that extracts entities, concepts, and relationships from conversation messages and renders them as an interactive force-directed Canvas graph. Fully heuristic — no API calls required.

Entity Extraction Heuristics

  • 🏷️Capitalised multi-word phrases (proper nouns)
  • 💻Code/tech terms: backtick-wrapped, camelCase, PascalCase, UPPER_CASE
  • 💬Quoted terms from the conversation
  • 📊Frequent non-stopword terms via TextAnalytics TF-IDF
  • 🔗Co-occurrence within same message creates relationship edges

Interaction

The graph supports drag to reposition nodes, scroll to zoom, click to select and highlight connections, and a search filter to find specific entities. Nodes are colour-coded by category and sized by mention frequency.

Access

Keyboard: Alt+Shift+K  |  Slash: /knowledgemap  |  Command Palette: "knowledge map"

⏰ Smart Deadline Tracker

AutonomousAlt+D

Autonomously scans conversation messages for dates, deadlines, and time references. Displays an interactive panel with sorted deadlines, live countdown timers, colour-coded urgency indicators, and proactive toast alerts when deadlines approach.

Detection Capabilities

  • 📅Explicit dates: "by March 15", "due Friday", "deadline: 2025-04-01"
  • ⏳Relative references: "in 2 hours", "by end of day", "next Monday"
  • 🏷️Context extraction: attaches the surrounding sentence as a label
  • 🔔Proactive alerts: toast notification when a deadline is approaching

Panel Features

Deadlines are sorted by urgency. Each entry shows a live countdown, the original message snippet, and colour-coding: green (plenty of time), yellow (approaching), red (imminent or overdue). Deadlines can be dismissed or snoozed individually.

🔍 Smart Response Critic

AutonomousAlt+Shift+Q

Analyses AI response quality using heuristic pattern matching. Scores each response on specificity, actionability, and confidence. Highlights hedging language, detects lack of concrete examples, and suggests when to push for better answers.

Analysis Dimensions

DimensionWhat It MeasuresDetection Method
HedgingUncertain language ("probably", "might be", "I think")Regex pattern matching against 17 hedging phrases
SpecificityConcrete data: percentages, proper nouns, code blocks, URLsRegex for numbers, capitalised phrases, backtick blocks, links
ActionabilityStep-by-step instructions, commands, code examplesPattern matching for numbered steps, code, shell commands, action verbs

API

SmartResponseCritic.toggle()            // open/close critic panel
SmartResponseCritic.analyze(msgIndex)   // analyse a specific response

⌨️ Smart Input Predictor

Autonomous

Learns from your typing patterns and offers ghost-text autocomplete suggestions. As you type (after 3+ characters), the predictor shows a faded continuation based on previously accepted phrases. Press Tab to accept.

How It Learns

  • 📝Records sent messages and builds a phrase frequency map (up to 50 phrases)
  • 🔍Matches current input prefix against learned phrases with startsWith
  • 👻Shows ghost text inline in the input field, styled with reduced opacity
  • 📊Tracks acceptance rate: suggestions shown vs. suggestions accepted

Configuration

SettingDefaultDescription
Max learned phrases50FIFO eviction when limit reached
Debounce300msWait time after keystroke before showing suggestion
Min characters3Minimum input length to trigger predictions

🔄 Smart Auto-Continue

AutonomousAlt+Shift+U

Detects truncated AI responses and either prompts the user or automatically sends a continuation request, merging the result seamlessly into the original response. Eliminates the manual "please continue" workflow.

Truncation Detection Signals

  • 🔓Unclosed code blocks (opening ``` without a matching close)
  • ✂️Mid-sentence cutoffs (ends without terminal punctuation)
  • 💬Explicit continuation markers ("continued in next message", "...")

Modes

ModeBehaviour
Manual (default)Shows a prompt asking the user to confirm continuation
AutomaticImmediately sends a continuation request without user input

Safety

A max chain limit of 5 prevents infinite continuation loops. Statistics (detected, continued, merged) are tracked and visible in the panel. A re-entrancy guard prevents concurrent detection during an active continuation.

🎙️ Voice Chat Mode

VoiceAlt+V

A hands-free conversational loop that creates a phone-call-like experience. Chains voice input → auto-send → wait for response → text-to-speech → auto-listen in a continuous cycle. Toggle with Alt+V or the floating mic button.

Loop Phases

  1. Listening — VoiceInput captures speech and auto-fills the chat input
  2. Sending — Message is automatically submitted via ChatController
  3. Waiting — Waits for the AI response to complete
  4. Speaking — ReadAloud speaks the assistant's reply
  5. Loop — When speech ends, auto-listen begins again

Requirements

Requires both VoiceInput (Web Speech API) and ReadAloud (Speech Synthesis API) to be available in the browser. A floating overlay shows the current phase, and users can pause/resume the loop or exit entirely.

✨ Prompt Enhancer

Alt+E

Sends your draft prompt to the AI with instructions to improve it, showing the result in a panel with multiple enhancement modes and a diff view. Use it, discard it, or tweak the mode.

Enhancement Modes

ModeGoal
🔍 ClarityRemove ambiguity, make the prompt clearer and more specific
📋 DetailAdd specificity, context, constraints, and format expectations
✂️ ConciseShorter and more direct without losing meaning
🧑‍🔬 ExpertRewrite as a domain expert would phrase it, with precise terminology
🎨 CreativeMore creative, open-ended, encouraging novel approaches

Workflow

Type a prompt → press Alt+E → pick a mode → preview the enhanced version → see a diff of changes → click "Use" to replace or "Discard" to keep the original. An abort controller cancels in-flight API requests when switching modes.

🔗 Session Linker

AutonomousAlt+L

Discovers related sessions by computing TF-IDF cosine similarity across all saved conversations. Shows a floating "Related Sessions" panel with ranked links and shared-topic badges. Proactively surfaces connections the user might otherwise miss.

Algorithm

  • 📄Extracts all user + assistant text from each session's messages
  • 📊Builds a TF-IDF index across all sessions (cached for 30 seconds)
  • 📐Computes cosine similarity between the current session and all others
  • 🏅Ranks results; filters below MIN_SIMILARITY = 0.08; shows top 8

API

SessionLinker.toggle()               // open/close related sessions panel
SessionLinker.show()
SessionLinker.hide()
SessionLinker.analyzeMessage()       // re-analyse after new message
SessionLinker.analyzeConversation()  // full session similarity report
