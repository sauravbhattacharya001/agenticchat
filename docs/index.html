Agentic Chat

A lightweight, zero-dependency browser chat interface that converts natural language prompts into executable JavaScript via OpenAI GPT-4o.

Overview

Agentic Chat is a single-page application that runs entirely in the browser. You describe what you want in plain English, and the AI agent writes and executes JavaScript code to accomplish it — all within a secure sandboxed iframe.

🧠 AI-Powered Code Generation

Uses OpenAI GPT-4o to translate natural language into working JavaScript code, then runs it automatically.

🔒 Sandboxed Execution

All generated code runs in an isolated iframe with strict CSP. Your page and data remain safe.

🔑 Per-Service API Keys

When generated code needs external API keys, the app prompts you and caches keys per domain.

📊 Token Tracking

Real-time token usage and cost estimation displayed after every interaction.

📦 Zero Dependencies

Pure HTML, CSS, and vanilla JavaScript. No build step, no bundler, no framework.

⚡ Conversation Memory

Maintains conversation context with automatic trimming to stay within token limits.

📋 Prompt Templates

Quick Start

No installation needed. Just open the HTML file or visit the deployed GitHub Pages site.

Option 1: GitHub Pages

Visit sauravbhattacharya001.github.io/agenticchat

Option 2: Local

git clone https://github.com/sauravbhattacharya001/agenticchat.git
cd agenticchat
# Open index.html in your browser — that's it!
open index.html

Usage

  1. Enter your OpenAI API key in the key field
  2. Type a natural language prompt (e.g., fetch the current Bitcoin price)
  3. The agent generates JavaScript, which runs in a sandbox
  4. Results appear in the console output area

Data Flow

User Input → ChatController.send()
  → ConversationManager.addMessage('user', prompt)
  → ChatController.callOpenAI(key, messages)
  → Parse response for code blocks
  → ApiKeyManager.substituteServiceKey(code)
  → SandboxRunner.run(code)
  → UIController.setConsoleOutput(result)

Orchestrates sending messages to OpenAI, processing responses, extracting code blocks, and coordinating between ConversationManager, SandboxRunner, UIController, and ApiKeyManager. This is the central "glue" module.

MethodReturnsDescription

KeyboardShortcuts

Global keyboard shortcut handler with a help modal (? key). Registers document-level keydown listeners for common actions like send, clear, toggle panels, and focus input.

CrossTabSync

Prevents multi-tab data corruption by detecting cross-tab localStorage writes via storage events and BroadcastChannel. Shows a non-intrusive banner offering Reload / Keep mine options when a conflict is detected. Uses a unique tab ID to distinguish own writes from foreign ones.

CostDashboard

Persistent API spend tracker with cumulative totals, per-model breakdowns, and budget alerts. Warns when spending crosses a user-set threshold. Data persists across sessions via localStorage.

PersonaPresets

Switchable system-prompt presets with custom persona support. Users can pick from built-in presets or define a custom system prompt that is injected as the first message in every conversation.

ModelSelector

Model picker dropdown with localStorage persistence. Lets users switch between available OpenAI models (configured in ChatConfig.AVAILABLE_MODELS) and remembers the last selection across sessions.

FileDropZone

Drag-and-drop file input for including file contents in chat prompts. Supports text-based files (.txt, .json, .csv, .js, .py, .md, .html, .css, .xml) up to 100 KB. Dropped file content is inserted into the input area.

FocusMode

Distraction-free zen mode toggled with Ctrl+Shift+F. Hides all secondary panels and UI chrome, leaving only the chat input and message area visible.

InputHistory

Navigate previous prompts with ↑/↓ arrow keys. Stores sent prompts in memory and lets users cycle through them without losing the current draft.

Scratchpad

Persistent notepad panel with copy, insert-into-chat, and download actions. Content persists across sessions. Useful for drafting prompts or saving intermediate notes.

ResponseTimeBadge

Displays a response-time indicator below the token usage area after each API call. Shows elapsed time in seconds or milliseconds to help users understand model latency.

ConversationFork

Branch conversations from any message into a new session. Copies the conversation up to the selected message into a fresh session, allowing users to explore alternative conversation paths without losing the original.

QuickReplies

Contextual follow-up suggestion chips shown after AI responses. Analyzes the latest response to generate 2–4 relevant follow-up prompts that users can click to continue the conversation.

MessagePinning

Pin important messages to a floating quick-jump bar at the top of the chat. Pinned messages persist across page reloads and can be clicked to scroll directly to the original message.

ReadAloud

Text-to-speech for messages using the Web Speech Synthesis API. Provides voice selection, speed controls, and a play/pause/stop interface. Supports reading individual messages or continuous playback.

MessageDiff

Compare any two messages with a visual line-level diff view. Highlights additions, deletions, and changes between selected messages. Useful for comparing model outputs across re-sends or forks.

ConversationTimeline

Visual minimap sidebar for conversation navigation. Renders a compact vertical timeline of all messages with color-coded role indicators. Click any point on the timeline to scroll to that message.

ConversationSummarizer

Heuristic conversation summary generator. Analyzes the current conversation to extract topics discussed, key decisions made, and action items identified — all computed client-side without additional API calls.

MessageAnnotations

Private notes and annotations on individual messages. Users can attach labeled notes (e.g., "important", "follow-up", "reference") to any message. Annotations persist in localStorage and are displayed inline with click-to-scroll navigation and inline note editing.

ConversationChapters

Named section dividers that split long conversations into navigable chapters. Includes a table-of-contents overlay for quick jumps to any chapter heading.

ConversationTags

Colored tag labels on saved sessions with filtering and management. Tags appear in the session list and can be used to categorize, filter, and organize conversations.

FormattingToolbar

Markdown formatting buttons above the chat input area. Provides one-click insertion of bold, italic, code, code block, heading, list, and link markdown syntax around the current selection or at the cursor position.

GlobalSessionSearch

Full-text search across all saved sessions (not just the current conversation). Returns matching sessions with highlighted snippets, letting users find past conversations by content.

AutoTagger

Heuristic topic detection and automatic tag suggestions. Analyzes conversation content to suggest relevant tags (e.g., "python", "debugging", "data-analysis") when saving sessions. Works entirely client-side using keyword matching and frequency analysis.

ResponseRating

Thumbs up/down ratings on AI responses with a model satisfaction dashboard. Tracks ratings per model over time and shows aggregate satisfaction scores, helping users identify which models perform best for their use cases.

ConversationMerge

Combine two or more saved sessions into one merged conversation. Interleaves messages chronologically based on timestamps, preserving the original ordering from each session.

ConversationReplay

Message-by-message playback of conversations with transport controls (play, pause, speed, skip). Useful for reviewing past conversations at a controlled pace or demonstrating workflows.

PromptLibrary

User-created prompt snippets organized into folders with search, usage tracking, and import/export. Unlike PromptTemplates (which ships built-in prompts), PromptLibrary is entirely user-curated with persistent storage.

MessageTranslator

Inline message translation to 20+ languages via the OpenAI API. Adds a translate button to each message that sends the content through the API for translation. Requires a valid API key.

MessageEditor

Edit and resend user messages. Truncates the conversation back to the edited message, reloads its content into the input area, and lets the user modify and re-send. Enables iterative prompt refinement without manual copy-paste.

ReaderView

Full-width reading overlay for long messages. Opens a clean, distraction-free view with adjustable font size and max-width. Triggered from the message context menu or keyboard shortcut.

TextExpander

Shorthand trigger system (Ctrl+Shift+X to manage). Define abbreviations like ;sig that auto-expand into full text when typed. Triggers and expansions are persisted to localStorage.

SmartScroll

Floating jump-to-latest button that appears when scrolled up, with an unread message count badge. Remembers scroll position per session so you return to where you left off.

PromptABTester

Side-by-side prompt comparison tool (Ctrl+Shift+B). Send the same prompt to two models (or the same model twice) and view responses in a split pane for quality evaluation.

PomodoroTimer

Built-in focus timer with configurable work/break intervals. Tracks completed sessions and provides notifications when cycles end. Integrates with the session streak tracker.

SmartPaste

Intercepts paste events and auto-detects content type — JSON gets pretty-printed, code gets fenced, CSV becomes a table, URLs get linked, and stack traces get formatted. Preserves raw paste with Ctrl+Shift+V.

SessionTemplates

Save reusable session configurations (system prompt, model, persona, settings) as named templates. Load with Ctrl+Shift+N for quick session setup. Templates are persisted and can be exported.

ConversationFlashcards

Extract question-answer pairs from conversations into flashcards for review. Supports manual card creation and basic spaced-repetition scheduling.

SmartTitleGenerator

Heuristic title generation from conversation content. Auto-titles sessions based on key topics and first messages, replacing generic "Session N" names.

PreferencesPanel

Centralized settings panel (Ctrl+,) that aggregates all configuration options in a single tabbed UI. Changes are applied live and persisted.

DraftRecovery

Configuration

All configuration lives in the ChatConfig object at the top of app.js. The active model can be changed at runtime via the model selector and is persisted to localStorage. Other constants can be modified in the source:

// In ChatConfig IIFE (app.js)
const _cfg = {
  _model: localStorage.getItem('ac-selected-model') || 'gpt-4o',
  MAX_TOKENS_RESPONSE: 4096,
  MAX_HISTORY_PAIRS: 20,
  MAX_INPUT_CHARS: 50000,
  SANDBOX_TIMEOUT_MS: 30000,
  AVAILABLE_MODELS: [
    { id: 'gpt-4o', label: 'GPT-4o' },
    { id: 'gpt-4o-mini', label: 'GPT-4o Mini' },
    // ...
  ],
  MODEL_PRICING: {
    'gpt-4o':        [2.50,  10.00],  // [input, output] USD per 1M tokens
    'gpt-4o-mini':   [0.15,   0.60],
    'gpt-4':         [30.00,  60.00],
    // ...
  },
  get MODEL() { return _cfg._model; },
  set MODEL(v) { _cfg._model = v; localStorage.setItem('ac-selected-model', v); }
};

Note: Static constants require a source edit and page reload. The model can be switched at runtime via the UI model selector or by setting ChatConfig.MODEL.

Security Model

Agentic Chat takes a defense-in-depth approach to executing AI-generated code:

Sandbox Isolation

  • Iframe sandbox: Code runs in an <iframe sandbox="allow-scripts"> — no access to parent DOM, cookies, or localStorage
  • CSP policy: default-src 'none'; script-src 'unsafe-inline'; connect-src https:
  • Origin isolation: Sandbox iframe has null origin, preventing same-origin attacks
  • Timeout: 30-second hard timeout prevents infinite loops

Communication

  • Nonce-based: Each execution uses crypto.randomUUID() to tag messages, preventing replay
  • Origin check: Only accepts postMessage from null origin (sandboxed iframes)
  • One-shot: Message handler removes itself after receiving the result

API Keys

  • Keys are stored in-memory only (never persisted to localStorage or disk)
  • Service keys are cached per-domain within the session
  • CI pipeline includes secret-scanning to prevent accidental commits
