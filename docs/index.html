Agentic Chat

A lightweight, zero-dependency browser chat interface that converts natural language prompts into executable JavaScript via OpenAI GPT-4o.

  2. Type a natural language prompt (e.g., fetch the current Bitcoin price)
  3. The agent generates JavaScript, which runs in a sandbox
  4. Results appear in the console output area

Architecture

The application is organized into six clearly separated modules, each implemented as an IIFE (Immediately Invoked Function Expression) exposing a clean public API:

ModuleResponsibility
ChatConfig Frozen configuration constants (model, limits, timeouts, system prompt)
ConversationManager Conversation history management â€” add, trim, clear, token estimation
SandboxRunner Iframe sandbox creation, code execution, timeout handling, cancellation
ApiKeyManager OpenAI key storage, per-service key prompting and caching
UIController All DOM manipulation â€” button states, output display, modal management

Configuration

All configuration lives in the ChatConfig frozen object at the top of app.js. To modify defaults:

const ChatConfig = Object.freeze({
  MODEL: 'gpt-4o',           // Change model here
  MAX_TOKENS_RESPONSE: 4096, // Max response length
  MAX_HISTORY_PAIRS: 20,     // Conversation depth
  MAX_INPUT_CHARS: 50000,    // Input limit
  SANDBOX_TIMEOUT_MS: 30000, // 30-second sandbox timeout
  SYSTEM_PROMPT: `...`       // Agent instructions
});

Note: Since the config is frozen, you must modify the source and reload â€” this is by design to prevent runtime tampering.

Security Model

Agentic Chat takes a defense-in-depth approach to executing AI-generated code:

Sandbox Isolation

  • Iframe sandbox: Code runs in an <iframe sandbox="allow-scripts"> â€” no access to parent DOM, cookies, or localStorage
  • CSP policy: default-src 'none'; script-src 'unsafe-inline'; connect-src https:
  • Origin isolation: Sandbox iframe has null origin, preventing same-origin attacks
  • Timeout: 30-second hard timeout prevents infinite loops

Communication

  • Nonce-based: Each execution uses crypto.randomUUID() to tag messages, preventing replay
  • Origin check: Only accepts postMessage from null origin (sandboxed iframes)
  • One-shot: Message handler removes itself after receiving the result

API Keys

  • Keys are stored in-memory only (never persisted to localStorage or disk)
  • Service keys are cached per-domain within the session
  • CI pipeline includes secret-scanning to prevent accidental commits
